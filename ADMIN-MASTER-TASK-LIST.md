## Customer Application — Complete Actions Inventory

The customer app performs the following write operations (all confirmed from source code):

| Action | API Endpoint | Tables Affected |
|---|---|---|
| Register (with optional referral code) | `POST /auth/customer/register` | `customers`, `users`, `referrals` |
| OTP verify | `POST /auth/customer/verify-otp` | `users` (verified flag) |
| Login / Logout | `POST /auth/customer/login` | `refresh_tokens`, `audit_logs` |
| Update profile | `PUT /customers/profile` | `customers`, `users` |
| Upload profile photo | `POST /customers/profile/image` | `customers` |
| Change / reset password | `PUT /customers/profile/password` | `users` |
| Activate preprinted card | `POST /customers/card/activate` | `cards` |
| Subscribe / save coupon to wallet | `POST /customers/coupons/:id/subscribe` | `coupon_subscriptions` |
| Unsave coupon from wallet | `DELETE /customers/coupons/:id/save` | `coupon_subscriptions` |
| Redeem coupon | `POST /customers/coupons/redeem` | `coupon_redemptions` |
| Accept gift coupon | `POST /customers/gift-coupons/:id/accept` | `gift_coupons` |
| Reject gift coupon | `POST /customers/gift-coupons/:id/reject` | `gift_coupons` |
| Add / remove favourite merchant | `POST/DELETE /customers/favourites/:id` | `customer_favourite_merchants` |
| Add / remove important day | `POST/DELETE /customers/important-days` | `customer_important_days` |
| File a grievance | `POST /customers/grievances` | `grievances` |
| Archive (close) own grievance | `PUT /customers/grievances/:id/archive` | `grievances` |
| Submit survey response | `POST /customers/surveys/:id/submit` | `survey_responses` |
| Participate in contest | `POST /customers/contests/:id/participate` | `contest_participants` |
| Apply to be a dealmaker | `POST /customers/dealmaker/apply` | `customers` (is_dealmaker flag) |
| Mark dealmaker task complete | `POST /customers/dealmaker/tasks/:id/complete` | `dealmaker_tasks` |
| Renew subscription | `POST /customers/subscription/renew` | `subscriptions` |
| Submit contact form | `POST /public/contact` | *(email/notification — no dedicated table)* |
| Submit business signup form | `POST /public/business-signup` | *(email/notification — no dedicated table)* |

**Read-only public browsing (customer app — no DB writes):**  
Advertisements · Flash discounts · Merchants + stores · Blog posts · Coupons + deals · Categories · CMS pages · Search

### Task 1 — Customer Grievance Management
**Status in previous list:** ✅ Valid — confirmed missing  
**Action:** **Expand scope** — customer app adds new dimensions

| Attribute | Detail |
|---|---|
| **Tables** | `grievances` |

**What is still missing in Admin (complete list):**
- List all grievances platform-wide — filterable by status, priority, merchant, city
- Detail view: full description + merchant resolution note + customer closure trace
- Override priority
- Override status (escalate open → in_progress, forcibly resolve or close)
- Reassign to specific admin (city/sales admin)
- Admin-post internal note (resolution_notes field)
- Dashboard KPI widget: unresolved grievances count by priority

**Priority:** 🔴 High  
**Task type:** New module (`GrievancesController` + views)

### Task 2 — Merchant Review Moderation
**Status in previous list:** ✅ Valid — confirmed missing  
**Action:** **Expand scope** — customer app is the source

| Attribute | Detail |
|---|---|
| **Tables** | `reviews` |
| **Data created by** | 👤 Customer (implied from `public.ts` — merchant detail shows avg_rating, total_reviews) |

| **Admin has** | Nothing |

**Customer app reveals:**
- `PublicMerchant` shows `avg_rating` and `total_reviews` — these are computed live from the `reviews` table
- Only **approved** reviews should count toward `avg_rating` — currently everything is `pending`, so all published rating metrics are incorrect
- Customer writes review against a `merchant_id` + optionally `store_id` + `rating` + `review_text`
- Merchant can reply but NOT approve/reject — admin is the sole gatekeeper

**What is still missing in Admin:**
- List all reviews with filter (status, merchant, rating range, date)
- View review detail including merchant's reply text
- Approve — makes review publicly visible and included in merchant's avg_rating
- Reject — hides review, sends notification to customer
- Delete — hard delete for extreme cases
- Bulk approve / reject (queue processing)
- Note: avg_rating display in customer app is directly broken until approvals are implemented

**Priority:** 🔴 High  
**Task type:** New module (`ReviewsController` + views)
