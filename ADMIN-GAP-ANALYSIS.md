## Section B — Missing & Incomplete Features

### B1. Advertisements / Banner Ads Management ❌ MISSING
**DB table:** `advertisements` (13 rows — title, media_type, media_url, link_url, display_duration, start/end_date, status, created_by_admin_id)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** Admins cannot create or manage banner ads shown in the customer/merchant apps. All 13 existing rows were manually seeded — no UI to update them.  
**Required operations:** List, Create, Edit, Delete, Toggle, Preview.

---

### B2. Blog / CMS ❌ MISSING
**DB table:** `blog_posts` (15 rows — title, slug, content, featured_image, author_id, status: draft/published/archived, published_at)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** Content editors cannot manage blog articles. All data is manually seeded.  
**Required operations:** List, Create (with rich editor), Edit, Publish/Archive, Delete. Slug auto-generation.

---

### B3. Audit Log Viewer ❌ MISSING
**DB table:** `audit_logs` (60 rows — user_id, user_type, action, table_name, record_id, old_values JSON, new_values JSON, ip_address, created_at)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** No admin can audit what changes were made, by whom, on which records. This is a security and compliance gap.  
**Required operations:** Read-only list with filters (user, action type, date range, table), detail view showing old/new JSON diff.

---

### B4. Customer Grievance Management ❌ MISSING
**DB table:** `grievances` (42 rows — customer_id, merchant_id, store_id, subject, description, status: open/in_progress/resolved/closed, priority: low/medium/high/urgent, resolved_at, resolution_notes)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** Admin has no way to view, assign, respond to, or escalate customer complaints. Grievances accumulate with no admin action possible.  
**Required operations:** List (with status/priority filters), Detail view, Update status, Add resolution notes, Assign priority, Close.

---

### B5. Merchant Review Moderation ❌ MISSING
**DB table:** `reviews` (60 rows — customer_id, merchant_id, store_id, rating 1–5, review_text, status: pending/approved/rejected)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** Reviews from customers pile up with `pending` status but no admin can approve or reject them. Merchants have no visible approved reviews on their profiles.  
**Required operations:** List (filter by status, merchant, rating), Approve, Reject, Delete.

---

### B6. Referral Tracking & Reward Management ❌ MISSING
**DB table:** `referrals` (32 rows — referrer_customer_id, referee_customer_id, referral_code, status: pending/completed/rewarded, reward_given, reward_amount, completed_at)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** Admin cannot view referral chains, manually trigger rewards, or investigate fraudulent referrals.  
**Required operations:** List (filter by status, customer, date), Detail view with referral chain, Manual reward trigger, Override status.

---

### B7. Gift Coupon Management ❌ MISSING
**DB table:** `gift_coupons` (22 rows — admin_id, customer_id, coupon_id, requires_acceptance, acceptance_status: pending/accepted/rejected, gifted_at, accepted_at, expires_at)  
**Admin UI:** None. No dedicated controller or view.  
**Impact:** Admins can gift coupons (per the DB data showing admin_id rows), but there is no UI to gift, view gifted coupons, track acceptance, or revoke gifts.  
**Required operations:** Gift a coupon to a customer, List all gifted coupons (filter by admin, customer, status), View acceptance status, Revoke pending gifts.

---

### B8. Subscription / Billing Management ❌ MISSING
**DB table:** `subscriptions` (49 rows — user_id, user_type: merchant/customer, plan_type: basic/standard/premium/monthly/quarterly/annual, start_date, expiry_date, auto_renew, status, payment_amount, payment_method)  
**Admin UI:** None. No controller, no view folder.  
**Impact:** Admin cannot view, create, renew, or cancel merchant or customer subscriptions. Cannot see expiring subscriptions or generate billing reports.  
**Required operations:** List (filter by type, status, expiry), Create/renew subscription for a user, Cancel/expire subscription, Export expiring subscriptions report.

---

### B9. Sales Registry ❌ MISSING (standalone module)
**DB table:** `sales_registry` (74 rows — merchant_id, store_id, customer_id, transaction_amount, transaction_date, payment_method, coupon_used, discount_amount)  
**Admin UI:** Partially surfaced in `ReportsController.redemptions()` but no dedicated CRUD view.  
**Impact:** Admin cannot browse, filter, or export individual transaction records. Coupon usage audit requires raw SQL.  
**Required operations:** List with date/merchant/store/customer/payment-method filters, CSV export, Coupon linkage view.

---

### B10. Store Gallery Management ❌ MISSING (standalone module)
**DB table:** `store_gallery` (119 rows — store_id, image_url, caption, display_order, is_cover, merchant_id)  
**Admin UI:** None. Images managed through merchant app only.  
**Impact:** Admins can't add/remove/reorder gallery images for a store, which is needed when onboarding a merchant manually.  
**Required operations:** List images per store, Upload, Set cover, Reorder, Delete.

---

### B11. Job Titles Master Data ❌ MISSING FROM MASTER DATA
**DB table:** `job_titles` (18 rows — job_title_name, profession_id, status)  
**Admin UI:** `MasterDataController` has `professions()` but no `jobTitles()` method.  
**Impact:** Job titles cannot be added, edited, or activated/deactivated. Titles are a child of professions and need to be managed together.  
**Required operation:** Add to `MasterDataController` as `jobTitles()` with full CRUD linked to parent profession.

---

### B12. Flash Discounts — Create & Edit ⚠️ INCOMPLETE
**DB table:** `flash_discounts`  
**Admin UI:** `FlashDiscountsController` has `index`, `detail`, `toggle`, `delete` — but **no `add()` or `edit()`**.  
**Impact:** Admins can only view/delete merchant flash discounts. They cannot create platform-level flash discounts or correct merchant errors.  
**Required operations:** Add `add()` and `edit()` methods with view forms.

---

### B13. Broadcasts — Full Implementation ⚠️ INCOMPLETE

**Required operations:** Compose broadcast (title, message, target: all/customers/merchants/specific city), Send, History list, Cancel scheduled.
